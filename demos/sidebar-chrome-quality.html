描边 chrome + 更静分区 + 略增行距，选中仍清楚但不吵。

C · 描边 + 静层级 推荐

在 B 的图标语言上，再压低分区标题、放宽行高、弱化未选中态。

  • 质感最接近 Linear 截图
  • 实现成本可控：侧栏专用 stroke 图标 + 少量 CSS token
  • 列表状态圆仍可用 Linear Issue 状态（业务语义另算）
选型建议：若目标是「侧栏像 Linear」，选 C；若只要图标统一、层级先不动，选 B。 不建议继续用现有实心 emoji 库当导航 chrome——那套更适合内容装饰，不是精密工具栏。 回复 A / B / C，我再写成设计稿并落实施计划。
