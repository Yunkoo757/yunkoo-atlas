侧栏优化 · 三套方案对照

当前侧栏只剩「工作台」4 项 + 底部回收站/设置，中间大片空白。 SECONDARY_NAV（进行中 / 星标 / 错过 / 模拟）与路由仍在，但 UI 未渲染。 下面三套方案都不改正式代码，仅供确认方向。

方案 A · 恢复快捷区 推荐起步

在一级导航与底部工具之间，加回「快捷」分区：进行中、星标、错过机会、模拟回测。沿用已有 SECONDARY_NAV / sidebarPins。

  • 改动小、立刻补上「模拟」入口
  • 填补空白，不推翻现有一级 IA
  • 风险：若继续往侧栏塞筛选，会重回臃肿

适用

你最在意「模拟回测」和几个高频状态入口常驻可见；筛选仍放在列表顶栏。

建议：先落地 A，快捷操作（导入/命令面板）用顶栏图标或侧栏底上一排小按钮，不必全做一级栏目。

方案 B · 固定视图 + 模拟一级

一级仍保留 4 个工作台模块；「模拟回测」升为与案例并列的一级（或紧挨工作台）。中间用「我的视图」展示已固定的保存视图（如导航3、本周亏损）。

  • 与已做的保存视图能力对齐
  • 侧栏内容随用户固定视图变化，空白被有意义地填满
  • 风险：未固定任何视图时仍偏空；需教用户「固定到侧栏」

适用

你希望侧栏反映个人工作流，而不是写死一堆系统状态。

可与 A 组合：系统快捷 2～3 项 + 我的固定视图。

方案 C · 模块内嵌套

点开「交易日志」展开子项（全部 / 进行中 / 星标 / 模拟）；「案例记录」展开重点/错题/待复看。更像 Linear Issue 的树。

  • 层级清晰，子项跟模块绑定
  • 交互更重，默认折叠时「模拟」仍可能被藏住
  • 风险：与顶栏快捷视图职责重叠，要仔细划界

适用

你希望侧栏承担「模块内导航」，顶栏少放芯片。

一般不如 A/B 直接；除非你明确要弱化顶栏快捷视图。
