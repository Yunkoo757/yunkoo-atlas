## 二、已完成的模块

### 2.1 数据模型扩展 ✅

**文件**: `src/data/case.ts`

**修改内容**:
```typescript
export interface CaseRecord {
  // ... 现有字段
  deletedAt?: string     // 删除时间（ISO 格式）
  deletedBy?: string     // 删除操作来源（可选）
}
```

**新增辅助函数**:
- `isDeleted(rec)` - 判断案例是否已软删除
- `isExpired(rec)` - 判断案例是否已过期（超过30天）
- `getRemainingDays(rec)` - 计算剩余天数

---

### 2.2 Store 层改造 ✅

**文件**: `src/store/useStore.ts`

**新增方法**:
- `removeCase(id)` - 软删除（设置 `deletedAt`）
- `restoreCase(id)` - 恢复（清除 `deletedAt`）
- `purgeCase(id)` - 彻底删除（从数组移除）

**修改逻辑**:
- 原硬删除改为软删除
- 保持 Undo/Redo 机制兼容性

---

### 2.3 视图层过滤调整 ✅

**修改文件**:
- `src/views/CaseList.tsx` - 添加 `.filter((c) => !c.deletedAt)`
- `src/views/DetailView.tsx` - 同上

**影响范围**: 所有案例列表视图正确过滤已删除数据

---

### 2.4 自动清理机制 ✅

**新增文件**: `src/lib/trashCleanup.ts`

**功能**:
- `cleanExpiredTrash()` - 清理过期案例及其关联资源
- `getTrashStats()` - 回收站统计信息

**集成点**: `src/App.tsx` 应用启动时自动执行清理

---

### 2.5 回收站 UI 开发 ✅

**新增文件**:
- `src/views/TrashView.tsx` - 回收站视图组件
- `src/views/TrashView.css` - 回收站样式

**功能特性**:
- 分组展示（即将过期、本周、本月、更早）
- 剩余天数倒计时显示
- 高亮即将过期数据（< 7天）
- 恢复按钮（绿色主按钮）
- 彻底删除按钮（红色次要按钮，需二次确认）

**路由**: `/trash` 已添加到 App.tsx

---

### 2.6 删除交互优化 ✅

**修改文件**:
- `src/views/CaseDetail.tsx` - Toast 提示改为"已移至回收站，30天后自动清空"
- `src/views/CaseList.tsx` - 移除确认弹窗，改为软删除
- `src/lib/caseMenu.tsx` - 右键菜单改为"移至回收站"

**用户体验改进**:
- 删除操作更流畅（无二次确认）
- Toast 提示更友好（告知30天机制）
- 可从详情页跳转到回收站查看

## 三、技术亮点

### 3.1 数据完整性保障

✅ **关联数据自然保留**
- 图片资源（`assets` 表）在恢复时自动可用
- 评论数据（`CaseComment[]`）内嵌保留
- 交易关联（`linkedTradeIds`）引用完整

✅ **清理机制可靠**
- 应用启动时自动清理过期数据
- 同步删除关联图片资源
- Console 日志记录清理过程

---

### 3.2 性能优化

✅ **查询性能保持**
- 过滤逻辑在前端执行（`.filter`）
- 无额外数据库查询开销
- 定期清理控制数组体积

✅ **存储空间管理**
- 30天后自动释放空间
- 孤儿图片资源同步清理
- 避免长期累积垃圾数据

---

### 3.3 兼容性设计

✅ **向后兼容**
- `deletedAt` 默认 `undefined` 表示未删除
- 旧版本数据自动视为"未删除"
- 无破坏性数据库迁移

✅ **Undo/Redo 保留**
- 软删除不破坏撤销栈
- 恢复操作可被撤销
- 与现有编辑体验一致
