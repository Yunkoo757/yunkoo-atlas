## 🎯 用户痛点

### 1. 字段过多，信息过载
- **问题**：新建窗口展示 15+ 个字段
- **影响**：用户心理负担重，犹豫不决
- **场景**：想快速记录一个交易机会，但被大量字段吓退

### 2. 必填字段不合理
- **问题**：策略、方向、信心度等需要立即填写
- **现实**：用户只想先截图，细节稍后补充
- **冲突**：强制填写 → 用户放弃记录

### 3. 图片上传体验差
- **问题**：截图需要手动点击上传按钮
- **期望**：直接粘贴到窗口，自动识别多张图片
- **现状**：拖拽上传支持有限

### 4. 时间字段不智能
- **问题**：开仓时间、平仓时间需要手动输入
- **期望**：自动填充当前时间或合理默认值
- **浪费**：每笔交易重复输入相同时间格式

---

## 💡 快速记录方案设计

### 方案 A：极简新建模式（推荐）

**核心思路**：
- **最小必填**：仅需选品种（Symbol）
- **自动填充**：ID、时间、状态、策略等自动生成
- **图片优先**：支持粘贴/拖拽多张截图
- **详情补充**：跳转到详情页再填写细节

### 1. Clipboard API 支持多图粘贴

```typescript
// 监听粘贴事件
window.addEventListener('paste', async (e) => {
  const items = e.clipboardData?.items
  if (!items) return

  for (const item of items) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile()
      if (file) {
        // 上传到 storage
        const assetId = await uploadAsset(file)
        // 添加到预览列表
        addImagePreview(assetId)
      }
    }
  }
})
```

## ⚠️ 需要权衡的点

### 1. 策略自动分配
- **问题**：自动分配到第一个策略可能不合理
- **方案**：
  - A: 默认"未分类"策略（需先创建）
  - B: 默认第一个策略（可在详情页修改）
  - C: 弹窗让用户选（但违背极简原则）

**推荐方案 B**：默认第一个策略，详情页可改

---

### 2. 方向默认值
- **问题**：默认 `long` 可能不符合实际
- **方案**：
  - A: 默认 `long`（详情页改）
  - B: 根据品种历史偏好（需统计）
  - C: 弹窗让用户选（违背极简）

**推荐方案 A**：默认 `long`，详情页改

---

### 3. 图片关联逻辑
- **问题**：粘贴的图片如何关联到交易？
- **方案**：
  - A: 存为交易笔记中的图片（TipTap 编辑器）
  - B: 存为独立图片资源，详情页引用
  - C: 同时支持两种方式

**推荐方案 C**：默认存为笔记图片，也支持独立资源
