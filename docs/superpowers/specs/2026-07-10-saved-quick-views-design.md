# 交易快捷视图与自定义筛选设计

日期：2026-07-10  
状态：已实施
适用模块：交易列表、案例记录列表

## 1. 目标

将交易列表顶部筛选栏中的空白区域升级为高频快捷视图栏，使用户能够：

1. 一键查看本周、本月、亏损交易和错题集。
2. 从“更多”菜单访问时间、结果、复盘、状态、时段和交易类型视图。
3. 将当前筛选组合保存为个人视图，并在后续一键恢复。
4. 在宽屏保持清晰可见，在窄屏自动收敛，不重新制造侧栏臃肿。

该功能不进入仪表盘统计逻辑，也不改变交易与案例记录的既有归属。

## 2. 设计原则

- **一键入口优先**：最高频视图直接显示，不要求打开筛选弹层。
- **预设与个人视图分离**：系统预设稳定，个人视图可编辑。
- **跨模块语义明确**：错题集、重点案例和待复看进入案例记录模块；亏损交易等保留在实盘交易列表。
- **不重复表达状态**：快捷视图已表达的条件不再重复渲染为活动筛选标签。
- **渐进披露**：常用项常驻，低频项进入“更多”，复杂组合由筛选器完成。
- **有限能力**：仅保存现有筛选器可表达的组合，不引入 AND/OR 条件构建器。

## 3. 快捷视图栏

### 3.1 桌面布局

快捷栏保持 `40px` 高度，按以下顺序排列：

1. 筛选按钮。
2. 内置常驻视图：本周、本月、亏损交易、错题集。
3. “更多”菜单。
4. 弹性空白。
5. “我的视图”标签。
6. 最多四个已固定个人视图。
7. 保存当前视图的 `+` 图标按钮。

内置视图与个人视图使用同一高度，但通过分区和文字颜色建立层级，不添加卡片或强阴影。

### 3.2 窄屏布局

- 常驻本周、本月和“更多”。
- 亏损交易、错题集及全部个人视图收进“更多”。
- 活动筛选条件继续以可移除标签显示。
- 工具栏不得产生页面级横向滚动。

### 3.3 激活状态

- 当前路由与筛选参数精确匹配某个视图时，该视图高亮。
- 同一视图表达的条件不重复显示为活动筛选标签。
- 在视图基础上叠加的额外条件仍显示为可移除标签。
- 再次点击当前活动视图不清空条件；清空通过“全部记录”或筛选弹层完成。

## 4. 内置视图

### 4.1 常驻视图

| 名称 | 行为 |
|---|---|
| 本周 | 进入本周实盘交易，按开仓日过滤 |
| 本月 | 进入本月实盘交易，按开仓日过滤 |
| 亏损交易 | 保留在实盘列表，筛选 `status=loss` |
| 错题集 | 跳转到 `/review-cases/mistakes` |

### 4.2 “更多”菜单

桌面端菜单按语义分组，省略已经常驻的入口；窄屏菜单补回被收起的常驻入口：

- 时间：今日、上周、上月。
- 结果：盈利、保本。
- 复盘：重点案例、待复看、已掌握。
- 状态：进行中、错过机会、星标交易。
- 时段：伦敦盘、纽约盘、亚盘、盘外时段。
- 类型：实盘、模拟、案例记录。

## 5. 个人保存视图

### 5.1 数据结构

```ts
type SavedTradeView = {
  id: string
  name: string
  pathname: string
  search: Record<string, string>
  pinned: boolean
  order: number
  createdAt: string
  updatedAt: string
}
```

`pathname` 保留模块上下文，`search` 保存筛选器当前支持的查询参数。首期支持：

- 时间范围或时间路由。
- 状态、品种、方向、策略。
- 交易时段。
- 普通标签和错误标签。
- 复盘分类与案例记录范围。
- 交易类型。

### 5.2 保存流程

1. 用户通过现有筛选器形成组合。
2. 点击 `+`。
3. 系统根据条件生成建议名称，例如“本月 · 伦敦盘 · 亏损”。
4. 用户确认或修改名称。
5. 当前固定数量少于四个时，新视图默认固定；已有四个时，新视图保存为未固定并进入“更多”。

没有任何有效筛选条件时，`+` 按钮禁用并通过提示说明原因。

### 5.3 管理操作

个人视图支持：

- 重命名。
- 拖动或菜单排序。
- 固定或取消固定。
- 删除。

内置视图不可修改或删除。

### 5.4 持久化与备份

- 保存到 Zustand 持久化快照。
- 纳入 JSON 导出与导入。
- 旧快照缺少该字段时归一化为空数组。
- 导入多个备份时按 `id` 合并，较新的 `updatedAt` 胜出。

## 6. 筛选能力补充

现有筛选器需要增加“交易时段”维度，复用已实现的结构化 `session` 字段与旧标签兼容识别逻辑。可选值包括：

- 伦敦盘。
- 纽约盘。
- 亚盘。
- 盘外时段。

筛选结果按交易的规范化时段类型匹配，而不是依赖显示文本或标签顺序。

## 7. 失效条件与异常处理

- 个人视图引用已删除的策略或标签时，视图继续保留。
- 打开该视图后，失效条件显示为可移除标签，并明确标记“已失效”。
- 失效条件不导致页面崩溃；其余有效条件继续应用。
- 保存名称去除首尾空白，不允许空名称。
- 同名视图允许保存，但在确认框中提示已有同名视图，避免误操作。
- 空结果沿用现有列表空状态，不显示错误提示。

## 8. 组件边界

- `QuickViewBar`：渲染内置视图、个人固定视图和活动状态。
- `QuickViewMenu`：展示分组预设与未固定个人视图。
- `SaveViewDialog`：生成建议名称并保存当前组合。
- `savedTradeViews` store slice：增删改、排序、归一化与持久化。
- `savedTradeView` helpers：路由规范化、精确匹配、建议名称和失效条件识别。

`TradeFilters` 继续负责筛选弹层与活动条件；快捷视图组件通过规范化路由对象与其协作，避免复制筛选逻辑。

## 9. 验收标准

1. 本周、本月、亏损交易可一键应用并正确高亮。
2. 错题集一键进入案例记录错题栏目。
3. “更多”菜单中的时间、结果、复盘、状态、时段和类型入口均可访问。
4. 任意支持的筛选组合可以保存、刷新后恢复、重命名、排序和删除。
5. JSON 备份可完整恢复个人视图。
6. 活动视图条件不重复显示，额外条件仍可单独移除。
7. 失效策略或标签不会导致崩溃。
8. 桌面端最多显示四个个人视图，窄屏不产生页面横向滚动。
9. 键盘焦点、按钮名称、菜单关闭和焦点返还符合现有可访问性标准。
10. 交易统计、案例记录统计和图片关联行为不受影响。

## 10. 测试范围

- 单元测试：规范化、匹配、建议名称、合并、失效条件、时段筛选。
- Store 测试：保存、重命名、排序、固定、删除和旧快照迁移。
- 路由测试：内置视图、跨模块错题入口、附加条件保留。
- 浏览器测试：保存流程、活动态、更多菜单、点击外部关闭、Escape 和焦点返还。
- 响应式测试：`1920px`、`1440px`、`900px` 和移动宽度无溢出。
- 回归测试：交易导入、图片隔离、仪表盘统计和案例记录排除规则。
