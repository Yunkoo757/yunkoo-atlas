# 去掉入场/出场价 + 详情「项目」改「策略」

日期：2026-07-17  
状态：待用户审阅规格  
相关规格：

- `2026-07-11-trade-detail-document-flow-design.md`（详情属性区）
- `2026-07-11-sidebar-navigation-redesign-design.md`（「我的空间」条目用语；本次不改）

## 1. 背景

复盘产品应以结果与过程为主。入场价、出场价要求用户核对图表点位，录入成本高、对统计与复盘价值低。

详情右侧属性区将策略选择标为「项目」，容易被理解成项目管理，而不是交易策略归属。

## 2. 目标

1. 主流程不再要求、展示或依赖入场价 / 出场价。
2. 平仓只通过盈亏金额、R 倍数（或保本）确认结果。
3. 详情属性区策略区块用语统一为「策略」，避免「项目」误导。
4. 旧数据与导入兼容：不破坏已有快照，不做破坏性迁移。

## 3. 非目标

- 不删除仓位、止损价字段，也不引入 `slPips`（可另开规格）。
- 不从 `Trade` 类型中删除 `entry` / `exit`（本次仅产品路径下线）。
- 不改左侧「我的空间」里「选择项目 / 搜索项目」等**工作区条目**用语（条目含保存视图与策略混排，不是策略实体本身）。
- 不改 CSV 列映射能力：导入仍可映射 entry/exit，但 UI 不再引导用户去填点位。

## 4. 设计决策

### 4.1 方案：UI + 产品路径一起砍（保留底层字段）

| 层 | 决策 |
|---|---|
| 详情 UI | 去掉「入场」「出场」行 |
| 平仓 | 删除 `resultMode === 'price'`（「出场价格」）及出场价输入 |
| 计算 | 主流程不再调用「按入出场价推导结果」的路径 |
| 数据模型 | `entry` / `exit` 暂留；新建/导入继续写默认 `entry: 0`、`exit: null` |
| 文案 | 去掉「请补充入场价、出场价」类引导 |

### 4.2 详情用语

| 位置 | 现文案 | 改为 |
|---|---|---|
| 详情属性区 section | 项目 | 策略 |
| 策略悬停预览副标题 | 策略项目 | 策略 |

## 5. 界面与交互

### 5.1 详情「交易数据」

保留（顺序可维持现状）：仓位、止损、盈亏金额、R 倍数（及既有结果编辑逻辑）。

移除：入场、出场。

点击改日期等既有行为不变。

### 5.2 平仓弹窗

- 结果模式仅保留手动填写（盈亏 / R / 保本），不再提供「出场价格」。
- 提交校验不再要求出场价；`prepareTradeClose` 的 `price` 分支从产品路径移除（实现时可删分支或改为不可达并测覆盖）。
- 可选填写的出场价输入一并删除。

### 5.3 导入与提示

- Notion / 列表帮助等文案：不再要求用户事后补入场价、出场价。
- CSV：列仍可映射；预览表头若仍显示「入场/出场」可保留为导入专用列名，或改为次要提示——以实现时最小改动为准，不以引导日常录入为目标。

## 6. 数据与兼容

- 已有交易中的 `entry` / `exit` 原样保留在存储中，只是 UI 不可见、不可编辑。
- 复制交易、导入、测试夹具可继续带这两个字段。
- 快照校验规则不收紧为「禁止字段」；也不要求清空历史值。

## 7. 测试要点

1. 详情交易数据区不出现入场、出场；仓位与止损仍可编辑。
2. 平仓弹窗无「出场价格」模式；用盈亏或 R 可正常平仓。
3. 既有含 entry/exit 的快照可加载，统计与复盘不因隐藏字段而报错。
4. 详情属性区 section 标题为「策略」；悬停副标题不为「策略项目」。
5. 左侧「我的空间」选择器仍使用「项目」条目用语（确认未被误改）。

## 8. 验收标准

- 日常建单 → 持仓 → 平仓 → 复盘，全程不出现入场/出场录入。
- 用户不会在详情把策略理解成「项目」。
- `main` 上现有相关单测更新后通过；无需数据迁移脚本。
