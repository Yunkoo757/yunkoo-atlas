# 自定义复盘笔记本与记录级复制粘贴设计规格

日期：2026-07-11  
状态：已确认

## 1. 背景

案例记录目前提供“全部、重点、错题、待复看、已掌握”等系统复盘视图，但用户还需要按个人经验组织长期专题，例如某段市场时期、某种心理状态、某类执行问题或任意自主主题。

这些主题不应被系统预设，也不应继续塞进标签或保存视图。系统只提供通用笔记本容器，名称、说明、正文和案例归纳完全由用户定义。

用户还需要接近 Notion 数据库行的记录级复制粘贴：在交易日志或笔记本中批量选择记录，通过 `Ctrl+C` 复制，再进入另一个笔记本或交易日志通过 `Ctrl+V` 生成独立副本。

## 2. 已确认的产品决策

1. 笔记本位于案例记录模块内部，不新增一级侧栏栏目。
2. 系统不预置笔记本、主题、模板、心流类型或自动归类规则。
3. 同一案例可以关联到多个笔记本。
4. 笔记本具有独立名称、简介和可编辑专题正文。
5. 笔记本默认按最近编辑排序，并支持手动置顶。
6. “加入笔记本”与“复制粘贴”是两种不同操作。
7. 加入笔记本只建立关联，不复制案例。
8. 复制粘贴生成独立副本；原记录与副本后续互不影响。
9. 交易日志粘贴到笔记本时生成案例记录，不进入仪表盘统计。
10. 笔记本记录粘贴到交易日志时生成实盘交易，并在执行前提示会影响统计。
11. 复制保留交易字段、复盘正文、截图、标签和来源关系，不复制评论、历史活动和删除状态。

## 3. 目标

- 让用户建立任意数量的自定义复盘笔记本。
- 让笔记本既能承载专题总结，也能组织现有案例记录。
- 支持交易日志、案例列表和笔记本之间的批量记录级复制粘贴。
- 保持交易、案例和笔记本的统计边界清晰。
- 保持图片资产、备份、导入导出和本地持久化完整。
- 保持现有 Linear 风格的紧凑、低噪声和可预期交互。

## 4. 非目标

- 不支持嵌套笔记本。
- 不提供系统笔记本模板。
- 不提供 AI 自动分类或推荐。
- 不提供跨设备剪贴板。
- 不提供多人协作、分享或权限控制。
- 不为笔记本增加独立统计报表。
- 不实现 Notion、Word 等外部应用的完整数据库导入协议。

## 5. 术语与行为边界

### 5.1 笔记本

用户定义的案例集合与专题文档。笔记本本身不进入仪表盘统计，也不改变所关联案例的复盘分类。

### 5.2 加入笔记本

将同一案例关联到一个或多个笔记本：

- 不创建新记录。
- 修改案例后，所有笔记本中显示最新内容。
- 从笔记本移除只解除关联。
- 删除笔记本不删除案例。

### 5.3 复制粘贴

以复制时的记录快照创建新记录：

- 新副本获得新的 ID、编号、收录时间和创建活动。
- 原记录和副本不再自动同步。
- 新副本保留直接来源关系，可跳回来源记录。
- 每次跨目标粘贴都遵循目标页面的记录类型。

## 6. 信息架构

案例记录顶部视图调整为：

```text
全部  重点  错题  待复看  已掌握  |  笔记本
```

“笔记本”是案例模块内的内容入口，不与案例复盘状态混为同一种筛选条件。它通过视觉分隔与前五个状态视图区分，但保持同一工具栏高度和控件语言。

确定路由：

- `/review-cases/notebooks`：笔记本列表。
- `/review-cases/notebooks/:id`：笔记本详情。

侧栏保持现有一级模块结构，不重复列出笔记本或笔记本内容。

## 7. 笔记本列表

笔记本列表使用紧凑行，不使用大卡片网格。每行包含：

1. 笔记本名称。
2. 一行简介。
3. 关联案例数量。
4. 最近编辑时间。
5. 置顶状态与行级菜单。

排序规则：

1. 置顶笔记本优先。
2. 同一置顶状态下按 `updatedAt` 倒序。

空状态只提供“新建笔记本”操作。系统不自动创建示例笔记本。

## 8. 笔记本详情

详情页分为两个连续区段：

### 8.1 专题正文

- 顶部显示可编辑名称和简介。
- 正文复用现有富文本编辑器。
- 支持标题、段落、清单、任务列表、引用、代码和图片。
- 图片继续使用现有持久化资产协议。
- 自动保存行为与交易复盘正文一致。

### 8.2 关联案例

- 下方复用现有紧凑交易行与批量选择能力。
- 默认按案例最近整理时间倒序。
- 支持现有品种、策略、状态、方向、时段和标签筛选。
- 支持“加入已有案例”“新建案例”和在空白区域粘贴记录。

## 9. 数据模型

新增实体：

```ts
export interface ReviewNotebook {
  id: string
  name: string
  description: string
  content: string
  pinned: boolean
  createdAt: string
  updatedAt: string
  archivedAt?: string
}
```

`Trade` 增加可选字段：

```ts
notebookIds?: string[]
sourceRecordId?: string
sourceRecordRef?: string
```

约束：

- 只有 `tradeKind === 'case'` 的记录可以长期关联笔记本。
- `notebookIds` 必须去重并过滤不存在的笔记本 ID。
- 删除笔记本时，从案例的 `notebookIds` 中移除对应 ID。
- 归档笔记本不解除案例关联，只从默认笔记本列表隐藏。
- 来源关系仅记录直接来源，不建立自动同步链。

## 10. 批量选择与操作栏

交易日志和案例列表沿用现有复选框选择模式。批量操作栏根据上下文显示：

### 10.1 交易日志

```text
已选 2 项 | 复制 | 删除
```

### 10.2 案例记录与笔记本

```text
已选 2 项 | 复制 | 加入笔记本 | 从当前笔记本移除 | 删除
```

不适用于当前上下文的命令不显示，不使用禁用按钮堆叠。

`Ctrl+C` 在存在列表选择时复制所选记录；没有列表选择时不拦截浏览器或编辑器原生复制。

## 11. 记录剪贴板协议

新增应用内 `RecordClipboard`，保存结构化记录快照：

```ts
export interface RecordClipboardBundle {
  version: 1
  bundleId: string
  copiedAt: string
  sourceContext: 'trade-log' | 'case-list' | 'notebook'
  sourceNotebookId?: string
  records: RecordClipboardItem[]
}

export interface RecordClipboardItem {
  sourceId: string
  sourceRef: string
  snapshot: Trade
}
```

复制时：

1. 将完整快照写入应用会话内存。
2. 尝试将带版本标记的结构化数据写入系统剪贴板。
3. 同时提供可读纯文本摘要，粘贴到外部应用时不会出现内部 JSON。
4. 系统剪贴板权限失败时，应用会话内复制粘贴仍可用，并显示弱提示。

粘贴时：

1. 优先读取应用结构化格式。
2. 校验协议版本和记录字段。
3. 根据目标上下文生成新记录。
4. 写入完成后清除当前列表选择并提示结果。

编辑器有文本选区或输入框获得焦点时，不拦截 `Ctrl+C`、`Ctrl+V`，继续使用原生内容复制粘贴。

## 12. 目标转换规则

### 12.1 交易日志到笔记本

- 每条来源交易创建一个新 `case` 记录。
- 使用新 `CAS-*` 编号。
- `notebookIds` 只包含目标笔记本。
- `recordedAt` 使用粘贴时间。
- 保留 `openedAt`、交易字段、复盘正文、截图、标签与错误标签。
- 写入 `sourceRecordId` 和 `sourceRecordRef`。
- 评论置空，活动只保留新的创建活动。
- 不进入仪表盘和账户统计。

### 12.2 案例或笔记本到笔记本

- 创建新的案例副本。
- 使用新 `CAS-*` 编号。
- 只关联目标笔记本，不继承来源的其他笔记本关系。
- 其余规则与交易日志到笔记本一致。

### 12.3 案例或笔记本到交易日志

- 粘贴前显示确认弹层，明确记录数量和统计影响。
- 确认后创建新的 `live` 记录与 `TRD-*` 编号。
- 不保留任何 `notebookIds`。
- 保留状态、盈亏和 R 倍数；新实盘记录会按照现有统计规则立即参与对应指标。
- 写入来源关系，重建创建活动。

### 12.4 交易日志到交易日志

- 创建新的 `live` 副本。
- 使用新 `TRD-*` 编号。
- 显示与案例转实盘相同的统计影响确认。

## 13. 图片资产规则

- 复制记录时不使用临时 `blob:` 地址作为持久化内容。
- 快照正文继续保存 `journal-asset://` 引用。
- 同一资料库内的副本可以共享底层资产 ID。
- 资产清理必须按全局引用计数处理；删除来源记录不得删除仍被副本或笔记本正文引用的图片。
- 笔记本正文图片必须进入备份资产收集范围。
- 缺失资产使用现有诊断占位，不阻断其他记录粘贴。

## 14. 重复粘贴保护

- 每个复制批次具有唯一 `bundleId`。
- 记录最近一次成功粘贴的 `bundleId + targetContext + targetId`。
- 在短时间窗口内重复触发同一组合时不再次创建记录，并提示“该批记录已粘贴”。
- 用户重新复制后生成新 `bundleId`，可以再次粘贴。
- 粘贴到不同笔记本视为不同目标，允许执行。

## 15. 键盘与焦点规则

- 列表存在选择且焦点不在编辑器、输入框或弹层中时，`Ctrl+C` 复制记录。
- 笔记本或交易列表获得页面级焦点时，`Ctrl+V` 执行记录粘贴。
- 编辑器、输入框、选择器和弹层内部始终保留原生剪贴板行为。
- 粘贴确认关闭后，焦点返回触发粘贴的列表容器。
- 批量栏按钮提供与键盘操作相同的行为和结果提示。

## 16. 错误处理

- 无有效记录负载时，页面级 `Ctrl+V` 不做任何操作，不弹错误。
- 剪贴板版本不支持时提示“记录来自不兼容版本”。
- 部分记录校验失败时，跳过失败项并汇总成功与失败数量。
- 目标笔记本不存在或已归档时拒绝写入。
- 转换到实盘前必须确认；取消不修改任何数据。
- 粘贴过程采用单次 Store 更新，避免只创建一部分记录后中断。

## 17. 持久化、导入导出与迁移

- `PersistedSnapshot` 增加 `reviewNotebooks`。
- Store 增加笔记本增删改、置顶、归档和案例关联操作。
- 存储 Schema 与导出版本从 `6` 升级到 `7`。
- 旧快照缺少 `reviewNotebooks` 时归一化为空数组。
- 旧交易缺少 `notebookIds` 和来源字段时保持兼容。
- JSON 与 `.journal.zip` 导入必须校验并合并笔记本。
- 合并规则按笔记本 ID 和 `updatedAt` 选择较新版本。
- 导出资产收集必须扫描交易正文、案例正文和笔记本正文。

## 18. 删除与归档

- 归档笔记本后，笔记本详情仍可通过直接链接访问并恢复。
- 删除笔记本需要明确确认。
- 删除只移除笔记本实体及案例关联，不删除案例。
- 删除后的笔记本正文不再参与资产引用；资产清理由全局引用检查决定。

## 19. 可访问性与视觉规范

- 笔记本列表行、案例行和批量栏沿用现有设计令牌。
- 不使用大卡片、渐变、装饰图形或独立配色主题。
- 所有图标按钮提供工具提示与 `aria-label`。
- 笔记本视图、列表、确认弹层和批量操作支持完整键盘流程。
- 选择、复制成功、粘贴成功和失败不能引发布局位移。
- 窄屏下批量栏必须保持主要操作可见，不出现横向页面溢出。

## 20. 测试策略

### 20.1 单元测试

- 笔记本归一化、排序、置顶和合并。
- 案例与多个笔记本的关联去重。
- 删除笔记本只解除关联。
- 四种来源与目标组合的记录转换。
- 新 ID、编号、时间和来源字段生成。
- 评论、历史活动、删除状态和来源笔记本关系不被复制。
- 重复粘贴窗口与不同目标判断。

### 20.2 存储测试

- 笔记本和关联关系刷新后保留。
- JSON 与 `.journal.zip` 导出导入后完整恢复。
- 笔记本正文图片与复制记录图片正确恢复。
- 删除来源记录后，副本图片仍可显示。

### 20.3 浏览器 QA

- 新建、重命名、置顶、归档、恢复和删除笔记本。
- 将同一案例加入多个笔记本。
- 交易日志多选复制并粘贴到笔记本。
- 笔记本多选复制并粘贴到另一个笔记本。
- 笔记本记录粘贴到交易日志的确认与取消。
- 编辑器获得焦点时不触发记录级粘贴。
- 桌面、窄屏和 Electron 下键盘流程可用。

## 21. 验收标准

1. 用户可以创建完全自定义的笔记本，不出现任何系统预设主题。
2. 同一案例可以加入多个笔记本且不产生副本。
3. 删除笔记本不会删除或修改案例正文。
4. 交易日志和笔记本均支持多选记录的 `Ctrl+C`、`Ctrl+V`。
5. 跨目标粘贴创建独立记录，并保留可追溯来源。
6. 案例副本不进入统计，实盘副本只有确认后才创建。
7. 复制后的截图刷新、备份恢复后仍属于正确记录。
8. 编辑器文本和图片的原生复制粘贴不受影响。
9. 所有新增流程通过单元测试、构建、浏览器 QA 和固定视口视觉检查。

## 22. 风险与控制

- **风险：记录复制与“加入笔记本”概念混淆。**  
  **控制：** 使用不同文案与入口；复制操作明确显示“创建副本”，加入操作明确显示“关联现有案例”。

- **风险：案例粘贴到交易日志后意外污染统计。**  
  **控制：** 所有实盘目标粘贴都必须经过包含数量和统计影响的确认。

- **风险：键盘事件抢占编辑器复制粘贴。**  
  **控制：** 只有列表存在选择或页面级目标明确时处理记录剪贴板；可编辑元素内永不拦截。

- **风险：复制记录后图片随来源删除而丢失。**  
  **控制：** 资产清理基于全局引用，副本与笔记本正文纳入引用扫描和导出收集。

- **风险：一次批量粘贴只完成部分记录。**  
  **控制：** 先完整校验与转换，再通过单次 Store 更新提交。
