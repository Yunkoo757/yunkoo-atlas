# 左侧导航与“我的空间”重设计规格

日期：2026-07-11  
状态：已确认，待实施计划  
相关规格：

- `2026-07-10-linear-frontend-rebuild-design.md`
- `2026-07-10-saved-quick-views-design.md`
- `2026-07-11-sidebar-quick-nav-design.md`

## 1. 背景

当前左侧栏包含四个固定一级模块与四个系统快捷入口。系统同时已经具备个人保存视图、策略页面、案例子视图、模拟回测路由，以及交易/案例工作区的最近页面记忆，但这些能力分别存在于侧栏、顶部快捷视图与路由状态中，尚未形成统一导航模型。

现状的主要问题不是入口数量不足，而是：

1. 系统入口与个人工作流割裂，保存视图无法进入侧栏。
2. 用户无法在一个位置管理常用目标的显示与顺序。
3. 核心模块、快捷入口和具体视图的页面切换职责不清。
4. 移动端把桌面导航压成横向滚动条，入口增长后难以使用。
5. 侧栏中间仍有大量无意义空白，但不能用堆叠筛选器来填充。

## 2. 目标

1. 保留稳定、可识别的四个核心模块。
2. 新增统一的“我的空间”，承载用户真实高频工作流。
3. 支持系统快捷、保存视图、策略、案例子视图和模拟回测的混排、自定义与溢出管理。
4. 明确核心模块“恢复现场”和自定义入口“精确直达”的不同切换语义。
5. 建立桌面、窄屏和移动端一致的数据模型与交互规则。
6. 复用现有保存视图、筛选、路由、计数和工作区记忆能力，不复制业务逻辑。

## 3. 非目标

- 不允许隐藏、删除或重排四个核心模块。
- 不支持用户创建多层文件夹或任意命名分组。
- 不把时间、方向、品种、标签等筛选维度直接铺回侧栏。
- 不允许固定设置子页、回收站等低频工具页面。
- 不新增复杂的条件构建器、跨页面自动推荐或动态最近访问排序。
- 不改变交易、案例、模拟数据的统计边界。

## 4. 设计原则

### 4.1 稳定核心，个性外围

今日记录、交易日志、案例记录、仪表盘固定存在，为用户提供稳定方向感。“我的空间”只优化个人效率，不承担应用一级信息架构。

### 4.2 一个强选中项

任何页面只出现一个强选中导航项。精确匹配“我的空间”条目时由该条目获得选中态；否则由相关核心模块获得选中态。

### 4.3 恢复现场与精确直达分离

- 核心模块：返回该模块最后使用的有效工作现场。
- “我的空间”：打开条目明确保存的精确目标，不混入最近状态。

### 4.4 引用而非复制

侧栏条目保存目标引用。显示名称、图标、路径和筛选条件从对应系统目标、保存视图或策略实时解析，避免产生第二套配置源。

### 4.5 有限常驻，渐进披露

“我的空间”最多常驻 8 项，其余用户已添加项目进入“更多”。可添加但尚未加入的目标只在管理选择器中出现。

## 5. 信息架构

### 5.1 桌面侧栏

自上而下：

1. 工作区头：头像、名称、搜索、新建。
2. 工作台：四个固定核心模块。
3. 我的空间：最多 8 个常驻自定义条目。
4. 添加或管理入口。
5. 弹性空白。
6. 回收站、设置。

固定核心模块：

| 模块 | 默认目标 | 现场记忆 |
|---|---|---|
| 今日记录 | `/today-record` | 路径、查询、列表模式 |
| 交易日志 | `/list` | 路径、查询、列表模式 |
| 案例记录 | `/review-cases` | 路径、查询、列表模式 |
| 仪表盘 | `/dashboard` | 不记忆子视图，固定默认页 |

### 5.2 “我的空间”可添加目标

| 类型 | 示例 | 目标来源 |
|---|---|---|
| 系统快捷 | 进行中、星标交易、错过的机会、模拟回测 | `SECONDARY_NAV` |
| 保存视图 | 导航3、本月伦敦盘亏损 | `savedTradeViews` |
| 策略 | ORA 模型、纽约开盘策略 | `strategies` |
| 案例子视图 | 重点、错题、待复看、已掌握 | 案例工作区内置视图 |

同一目标不得重复加入。

### 5.3 移动端

移动端不再使用横向滚动侧栏，改为固定底部五项：

1. 今日
2. 交易
3. 案例
4. 仪表盘
5. 更多

“更多”打开底部抽屉，包含：

- 我的空间，顺序与桌面一致。
- 搜索。
- 设置。
- 回收站。
- 管理我的空间。

移动端管理使用独立全屏列表，不在底部小抽屉内执行拖拽。

## 6. 数据模型

```ts
type SidebarTarget =
  | { kind: 'system'; id: SidebarNavId }
  | { kind: 'saved-view'; viewId: string }
  | { kind: 'strategy'; strategyId: string }
  | { kind: 'case-view'; scope: ReviewCaseScope }

type SidebarWorkspaceItem = {
  id: string
  target: SidebarTarget
  placement: 'pinned' | 'overflow'
  order: number
}
```

持久化位置确定为 `display.sidebarWorkspaceItems`。条目不保存目标名称或完整路由副本。

### 6.1 常驻与溢出

- `placement: 'pinned'` 最多 8 项。
- 用户添加第 9 项及以后时，默认使用 `overflow`。
- 导入或旧数据出现超过 8 个 `pinned` 时，按规范化后的顺序保留前 8 项，剩余项迁移为 `overflow`。
- “更多”展示全部 `overflow` 条目，并提供进入管理模式的入口。

### 6.2 目标解析

- 系统快捷从 `SECONDARY_NAV` 解析当前文案、图标和路由。
- 保存视图从 `savedTradeViews` 解析当前名称、规范化路径和查询条件。
- 策略从 `strategies` 解析当前名称、图标和 `/strategy/:id` 路径。
- 案例子视图从案例工作区内置视图解析名称和路径。
- 显示计数必须复用现有 `filterTrades` 与对应页面相同的统计口径。

## 7. 页面切换逻辑

### 7.1 核心模块：恢复现场

今日、交易和案例分别保存最后一次有效的：

- `pathname`
- `search`
- 列表 / 看板 / 表格模式（由规范化路径承载）

点击核心模块时，优先进入其最近有效现场；没有记忆或目标失效时进入该模块默认页。

仪表盘始终进入 `/dashboard`。

### 7.2 “我的空间”：精确直达

点击条目时实时解析其目标：

- 系统快捷进入当前系统路由。
- 保存视图进入其最新 `pathname + search`。
- 策略进入对应策略页。
- 案例子视图进入固定案例范围。

该过程不合并核心模块的最近搜索条件。

### 7.3 激活态优先级

1. 规范化当前位置。
2. 查找精确匹配的“我的空间”条目。
3. 若匹配，则仅该条目强选中。
4. 若不匹配，则根据工作区归属选中对应核心模块。

当当前位置基于某个固定视图又叠加了临时筛选时：

- 固定视图继续保持选中。
- 条目旁显示一个弱小圆点，表示“当前条件已修改”。
- 再次点击该条目恢复其原始条件。

### 7.4 详情页返回

从列表、看板或表格打开交易详情时，导航状态保存：

- 来源 `pathname + search`
- 来源列表模式
- 来源滚动位置或稳定滚动锚点

关闭详情或点击返回时，优先恢复来源现场；来源失效时回退到该记录所属模块默认页。

## 8. 管理交互

### 8.1 进入方式

- 点击“我的空间”标题右侧 `···`。
- 点击列表末尾“添加或管理”。

管理模式在侧栏原地展开，不跳转设置页。

### 8.2 编辑行为

- 四个核心模块显示“固定”，不可删除或排序。
- 自定义条目显示拖拽手柄和移除按钮。
- 拖拽即时预览顺序，点击“完成”后统一持久化。
- 移除后提供轻量撤销操作。
- `N / 8` 显示当前常驻数量。
- “恢复默认”恢复系统快捷默认组合，但必须在执行前明确说明会替换当前“我的空间”配置。

### 8.3 添加选择器

“浏览可添加项目”打开右侧选择面板，按以下类别展示：

1. 系统快捷
2. 我的视图
3. 策略
4. 案例视图

选择器支持搜索。已加入目标显示当前位置状态：常驻或更多。用户可以在常驻、更多、未添加之间切换。

### 8.4 键盘操作

- `Enter`：打开条目。
- 编辑模式下 `Alt + ↑/↓`：调整顺序。
- `Delete`：移除当前自定义条目。
- `Escape`：放弃未保存草稿并退出编辑。
- 编辑完成或取消后，焦点返回原管理入口。

## 9. 迁移与异常处理

### 9.1 旧数据迁移

现有 `display.sidebarPins` 按原顺序迁移为 `kind: 'system'` 的常驻条目。迁移成功后，新模型成为唯一写入源；旧字段只保留兼容读取周期，不再由新 UI 写入。

### 9.2 规范化

- 按目标语义去重，保留顺序靠前项。
- `/paper`、`/practice` 等别名统一解析为 `/sim`。
- 非法 `placement`、无效 `order` 和未知系统 id 使用安全默认值或丢弃。
- 超过常驻上限的条目迁移为 `overflow`，不静默删除。

### 9.3 目标失效

保存视图或策略被删除后：

- 日常侧栏和移动“更多”不显示失效入口。
- 管理模式保留该条目，标记“已失效”。
- 用户可删除失效项或重新选择目标。
- 失效条目不得阻止其余条目解析和渲染。

### 9.4 工作区记忆失效

若记忆路径不再属于目标模块、引用了不存在的资源或无法规范化：

1. 回退到模块默认页。
2. 清除该模块损坏的记忆。
3. 不显示错误页面或阻断导航。

### 9.5 未保存编辑

编辑草稿只存在于组件本地状态。点击完成前刷新、关闭或离开页面时放弃草稿，不写入持久化快照。

## 10. 组件与职责边界

### `Sidebar`

负责桌面布局、核心模块、“我的空间”常驻项、激活态和管理模式入口，不直接实现目标解析规则。

### `MobileNavigation`

负责移动底栏与“更多”抽屉，消费同一套核心模块和“我的空间”解析结果。

### `SidebarWorkspaceEditor`

负责草稿、排序、移除、撤销、常驻上限和完成/取消。

### `SidebarTargetPicker`

负责按类别搜索和选择可添加目标。

### `sidebarWorkspace` helpers

负责规范化、迁移、去重、排序、目标解析、失效识别、精确匹配与激活态优先级。

### Store

负责持久化最终 `sidebarWorkspaceItems`，提供替换配置的单一动作。编辑过程中不逐步写入 Store。

## 11. 可访问性与视觉规则

- 延续现有 244px 桌面侧栏、28–30px 紧凑导航节奏。
- 只有一个强选中背景，不使用品牌色大面积填充。
- “已修改”状态同时使用圆点和可访问文本，不只依赖颜色。
- 图标按钮必须提供可访问名称和工具提示。
- 拖拽必须提供完整键盘替代操作。
- 移动端底栏与抽屉操作命中区至少 44px。
- 不产生页面级横向滚动。
- 编辑模式切换不得改变侧栏宽度。

## 12. 验收标准

1. 四个核心模块固定存在，用户不能删除或排序。
2. “我的空间”支持四类目标（系统快捷含模拟回测）的混排、拖拽、移除和搜索添加。
3. 常驻最多 8 项，溢出项进入“更多”。
4. 核心模块恢复最近有效现场，自定义条目始终精确直达。
5. 任意页面只出现一个强选中项。
6. 临时叠加筛选时固定视图显示“已修改”，再次点击恢复原始条件。
7. 详情返回恢复来源路径、查询、列表模式与滚动位置。
8. 旧 `sidebarPins` 自动迁移且不丢顺序。
9. 删除保存视图或策略不会导致侧栏崩溃。
10. JSON 导入导出完整保留新配置，并正确规范化重复、别名和超限条目。
11. 移动端使用五项底栏和“更多”抽屉，不使用横向滚动侧栏。
12. 键盘排序、删除、取消和焦点返还可用。

## 13. 测试范围

### 单元测试

- 旧模型迁移。
- 目标语义去重。
- 常驻上限与溢出迁移。
- 路由别名规范化。
- 四类目标（系统快捷含模拟回测）解析与失效识别。
- 精确匹配、核心回退和“已修改”判定。

### Store 与持久化测试

- 完整替换配置。
- 导入导出与旧快照兼容。
- 顺序、placement 与失效条目保留。

### 路由测试

- 今日、交易、案例工作区现场记忆。
- 系统快捷、保存视图、策略和案例子视图精确跳转。
- 列表、看板、表格模式保持。
- 详情返回来源现场与失效回退。

### 组件与浏览器测试

- 进入/退出管理模式。
- 拖拽预览、完成持久化和 Escape 取消。
- 删除与撤销。
- 搜索添加、重复拦截、常驻/更多切换。
- 单一强选中态与“已修改”状态。
- 焦点返还和键盘排序。

### 响应式测试

- 桌面 1920px、1440px。
- 窄屏 900px。
- 移动 390px。
- 底栏、抽屉、全屏管理页无横向溢出，命中区满足 44px。

## 14. 实施顺序建议

1. 建立纯函数数据模型、迁移与测试。
2. 扩展工作区记忆与激活态解析。
3. 重构桌面侧栏消费统一模型。
4. 实现管理模式与目标选择器。
5. 实现移动底栏与“更多”抽屉。
6. 完成详情返回、响应式与浏览器验收。

本规格只确定设计与行为边界。具体文件修改、提交拆分和验证命令由后续实施计划确定。
