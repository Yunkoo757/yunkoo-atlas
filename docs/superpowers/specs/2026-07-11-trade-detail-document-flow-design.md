# 交易详情正文纯文档流设计

## 1. 背景

交易详情页当前会从正文提取图片，再额外渲染大图舞台与横向缩略图条，同时隐藏编辑器中的原图。这造成图片重复承担“正文内容”和“媒体浏览器”两种职责，正文区域视觉重量过高，也破坏原始图文顺序。

本次改造只为正文区域减重。右侧交易属性侧栏、顶部导航、交易数据和持久化协议保持不变。

## 2. 目标

- 将交易详情正文恢复为单一、连续的文档流。
- 图片只在富文本编辑器中的原始位置显示一次。
- 个人评论默认可见，便于第一时间查看自己的备注。
- 系统活动记录默认折叠，避免抢占正文空间。
- 保持现有编辑、自动保存、图片资产和灯箱能力。

## 3. 非目标

- 不修改右侧交易属性侧栏的结构或交互。
- 不重写富文本编辑器、图片存储或交易数据模型。
- 不迁移、复制或重新排列已有正文内容。
- 不改造列表、看板、表格或交易创建弹窗。

## 4. 页面结构

正文列按以下顺序组成：

1. 品种标题与方向标记。
2. 无卡片外壳的富文本编辑器。
3. 默认展开的个人评论列表与评论输入框。
4. 默认折叠的系统活动记录入口。

删除独立大图舞台、横向缩略图条和“暂无截图”占位。正文没有图片时不渲染任何媒体占位。

## 5. 编辑器与图片

- 继续复用现有 Tiptap 编辑器。
- 保留标题、段落、列表、任务清单、引用、代码和行内格式。
- 保留图片粘贴与拖入。
- 图片按正文中的原始节点顺序纵向显示，自适应正文宽度，不裁切。
- 单击图片维持编辑器选择行为；双击图片打开现有灯箱。
- 图片加载失败时，在原节点位置显示轻量错误占位，不影响其余正文。
- 移除为媒体舞台隐藏编辑器图片的样式规则。

## 6. 评论与活动

评论属于用户需要快速读取的个人备注，因此保持外露：

- 评论列表默认显示。
- 评论输入框紧随评论列表。
- 现有发送、删除与时间信息保持不变。

系统生成的创建、状态变化、策略变化和正文更新等活动收进“活动记录 · N”折叠入口：

- 默认收起。
- 点击后展开完整活动列表。
- 再次点击收起。
- 折叠不改变或删除任何活动数据。

## 7. 数据与保存

- 编辑器继续使用现有自动保存与保存状态反馈。
- 图片继续使用 `journal-asset://` 资产引用和现有显示解析流程。
- 已有交易正文无需迁移，打开后直接按新布局渲染。
- 灯箱继续从当前正文收集图片，并保持正文顺序。
- 本次改造不改变导入、导出、备份和资产清理行为。

## 8. 异常与边界

- 空正文只显示编辑器占位文字，不显示媒体空状态。
- 只有图片的正文直接显示图片。
- 多张图片按正文顺序自然纵向排列，不生成缩略图导航。
- 图片解析失败只影响对应图片节点。
- 没有评论时仍显示轻量评论输入框。
- 没有活动时不显示活动折叠入口。

## 9. 验收标准

1. 交易详情页不再出现独立大图舞台或横向缩略图墙。
2. 正文图片只出现一次，并保持原始图文顺序。
3. 图片宽度适配正文列且不被裁切。
4. 粘贴或拖入图片后可自动保存，刷新后仍显示在原位置。
5. 双击正文图片仍可打开灯箱，并能浏览同一正文中的其他图片。
6. 评论列表和输入框默认可见，评论操作保持可用。
7. 系统活动默认折叠，可展开和再次收起。
8. 右侧交易属性侧栏的桌面与窄屏行为不变。
9. 旧交易无需数据迁移即可正确展示。
10. 现有回归测试和生产构建通过。
