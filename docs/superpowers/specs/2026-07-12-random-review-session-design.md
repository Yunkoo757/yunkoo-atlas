# 随机复盘会话（A 抽卡 + B 沉浸）设计

日期：2026-07-12  
状态：待用户确认后进入实现计划  
相关 demo：`.superpowers/brainstorm/20260712-random-review/`（本地，不入库）

## 1. 目标

提供独立「随机复盘」模块：从**交易日志 + 案例记录**组成的统一池中洗牌抽取条目，以抽卡方式快速过题；需要时一键进入沉浸式图文阅读。会话内只读，编辑仍走现有详情页。

成功标准：

- 用户能在 10 秒内开始一轮会话，并连续浏览多笔历史记录（含图文笔记）。
- 抽卡与沉浸共用同一随机队列，切换模式不丢当前位置。
- 不引入新的持久化实体；不与 list/board/table 工作台搅在一起。

## 2. 范围

### 包含（V1）

- 侧栏一级入口「随机复盘」与路由 `/review-session`
- 会话开始筛选 + 洗牌队列
- **模式 A（默认）**：抽卡翻转（先回忆，再揭答案）
- **模式 B**：沉浸阅读（左图右文 / 窄屏上下），从 A 进入
- 池子默认包含：**全部案例 + 全部交易日志**（可在开始前关掉其中一类）
- 快捷键：Space 翻转；N / → 下一张；Esc 从 B 回 A（或退出确认，见下）
- 「打开详情」跳转现有 `/trade/:ref`（可编辑）

### 不包含（V1）

- Anki 式间隔重复 / 熟练度算法
- 会话内编辑笔记或改属性
- 牌堆双栏浏览（方向 C）
- macOS 应用内更新 / 公证等无关事项
- 跨设备同步「本轮进度」（进度仅存内存或 sessionStorage 即可）

## 3. 信息架构与导航

| 项 | 选择 |
|----|------|
| 入口 | `PRIMARY_NAV` 新增一项，建议 id `reviewSession`，label「随机复盘」，路径 `/review-session` |

## 4. 数据模型（不新增实体）

复用 `Trade`：

- 池内条目 = `trades` 中满足筛选条件的记录
- 案例：`tradeKind === 'case'`
- 交易日志：`tradeKind === 'live' | 'paper'`（与现有「账户交易」口径一致，具体过滤函数复用 `isAccountTrade` / 对称逻辑）
- 展示字段：`ref`、`symbol`、`side`、`pnl`（或 R，与列表一致）、`strategy`、`tags` / `mistakeTags`、`title`（可用 symbol+策略或 note 首行摘要）、`note`、`reviewCategory` / `reviewStatus`、`openedAt` / `recordedAt`
- 图片：`note` 内经 `resolveNoteForDisplay` 解析的 `journal-asset://`；沉浸模式提取 note 内 `<img>` 作左侧画廊，点击走现有 Lightbox

无笔记且无图的条目仍可进入池（正面信息足够）；翻转后显示空态文案「暂无复盘笔记」。

## 5. 会话与筛选

### 5.1 开始面板（进入路由后若无活跃会话）

默认勾选：

- [x] 案例记录
- [x] 交易日志（含实盘 / 模拟，与产品现有 live/paper 划分一致）

可选细分（与案例 scope 对齐，仅当「案例」勾选时生效）：

- 全部案例 / 错题 / 重点 / 待复看 / 已掌握

可选：排除 `note` 为空的条目（默认关闭，避免池子过小）。

点击「开始」→ 构建数组 → `shuffle` → `cursor = 0` → 进入模式 A。

### 5.2 队列规则

- 本轮内按队列顺序前进；「下一张」=`cursor++`
- 到末尾：提示「本轮结束」并提供「再洗一轮」（重新 shuffle，cursor=0）或「调整筛选」
- V1 **不**做「本轮已看过标记持久化」；刷新页面则会话丢失（可接受）。若实现成本低，可用 `sessionStorage` 暂存队列 id 列表与 cursor，同标签页刷新可恢复

### 5.3 随机性

使用 Fisher–Yates；种子不要求可复现（V1）。

## 6. 交互：模式 A（抽卡）

### 正面（未翻转）

- 种类徽章：案例 / 交易
- 分类徽章（若有）：错题 / 重点 / …
- 大号 `symbol`、标题、方向、策略、日期
- 盈亏（颜色语义与列表一致）
- 提示：「先回忆，再翻转」

**不**在正面展示笔记正文与截图（保证回忆空间）。

### 背面（已翻转）

- 缩略图区（取 note 首图或占位）
- 笔记 HTML 只读渲染（高度有上限，溢出滚动）
- 主操作：「下一张」「深读」
- 次操作：「再看正面」（取消翻转）

### 手势 / 键位

| 输入 | 行为 |
|------|------|
| 点击卡片 / Space | 切换翻转 |
| N / → | 下一张（若在正面，直接进下一张并重置为正面） |
| 按钮「深读」 | 进入模式 B，保持同一 cursor |

## 7. 交互：模式 B（沉浸）

- 布局：宽屏左视觉（图库 + 主图）/ 右笔记；窄屏上图下文
- 顶栏：返回抽卡、进度（如 `3 / 31`）、下一张、打开详情
- 笔记完整展示；图点击 → 现有 `ImageLightbox`
- 只读；「打开详情」→ `navigate(tradeDetailPath(trade))`，返回时理想情况回到会话（V1 可用 `location.state.from = '/review-session'`；若详情返回逻辑复杂，V1 允许回到列表，实现计划里对齐 `tradeDetailNavState`）

从 B「下一张」：cursor++，仍留在 B（连续深读）；「返回抽卡」：回 A 且该卡为正面未翻转。

## 9. 错误与空态

| 情况 | 处理 |
|------|------|
| 筛选后池为空 | 开始按钮禁用 + 说明「放宽筛选」 |
| 单条 note 解析失败 | 背面显示文本占位，不打断会话 |
| 资产图 404 | 占位块，不崩溃 |
| 用户在会话中点侧栏离开 | 直接离开；再次进入显示开始面板（V1 不打断确认，除非有未写脏数据——本模块无脏数据） |

## 10. 视觉

- 沿用 Atlas 暗色 tokens（`tokens.css`），与 demo A/B 同系
- 避免再引入第三套「游戏化」视觉；抽卡可用轻量翻转，不加强烈拟物
- 正面信息密度低于列表行，留白多于工作台

## 11. 测试要点

- 建池：默认两类都进；只关案例 / 只关交易时数量正确
- 案例 scope 筛选只作用于案例子集
- shuffle 后长度不变、无重复 id（单轮内）
- A↔B 切换 cursor 不变
- 到末尾再洗一轮
- note 含 `journal-asset://` 时在 B 能显示（Electron + web 适配路径与 DetailView 一致）
- 回归：侧栏四项变五项后，现有 pin / 命令面板不报错

## 12. 实现分期建议

1. **Slice 1**：路由 + 侧栏入口 + 建池/洗牌 + 模式 A（可先纯文本背面）
2. **Slice 2**：模式 B + 图片提取 + Lightbox
3. **Slice 3**：开始面板细分筛选、空态、sessionStorage 恢复、命令面板

## 13. 已决决策

| 决策 | 结论 |
|------|------|
| 主交互 | A 抽卡 |
| 深读 | B 沉浸展开 |

| 编辑 | 会话只读，跳转详情 |
