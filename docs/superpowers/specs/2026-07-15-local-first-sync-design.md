# 本地优先云同步设计

日期：2026-07-15  
状态：已确认方向，分阶段实施

## 目标

- Windows 与 macOS 始终保有可独立使用的完整本地资料库。
- 启动先读取本地数据，不以网络请求阻塞首屏；同步在后台立即开始。
- 断网期间所有记录与编辑能力可用，恢复网络后自动上传。
- 多设备并发修改不采用整库覆盖，也不静默丢弃冲突。
- 原图可选择全部保留离线，云端只承担连续性与设备间分发。

## 当前约束

- 当前 `StorageAdapter` 以整份 `PersistedSnapshot` 为持久化单位。
- Electron 使用 `sql.js` 导出并原子替换整个 `journal.db`。
- 附件以独立文件保存，尚无远端版本、内容哈希或下载状态。
- 交易、策略、标签等实体缺少统一 revision、deviceId 与删除墓碑。
- iCloud / OneDrive 同步正在使用的库目录只能产生文件级冲突，不能合并业务实体。

因此，云同步不得直接上传或同步正在使用的 `journal.db`。

## 选择

在现有本地存储模块上增加增量同步层，首选 Supabase 提供认证、Postgres、私有对象存储与实时唤醒；应用自行负责本地 outbox、幂等、冲突策略和客户端加密。

暂不直接迁移至 PowerSync。该方向可通过独立 PoC 验证 Electron 原生 SQLite 打包、Windows/macOS 双架构和旧资料库迁移后再决定。

## 数据流

```text
Renderer / Zustand
       ↓ 本地提交
Local Storage ── 同一事务 ── Snapshot + Outbox
       ↑                             ↓
  应用远端增量                  Sync Module
                                      ↓
                          加密操作日志 + 加密附件
```

任何用户修改都先可靠写入本机，再由后台同步。关闭应用不等待网络。

## 本地同步元数据

- `sync_state`：libraryId、deviceId、epoch、pullCursor、lastSyncAt。
- `sync_outbox`：opId、deviceSeq、entityType、entityId、baseRevision、payload、state。
- `entity_versions`：实体 revision、字段组版本、删除墓碑。
- `asset_sync`：sha256、objectKey、localState、remoteState、byteSize。

服务端为上传操作分配单调递增 cursor；实时通道只唤醒拉取，不作为数据真相。

## 同步边界

跨设备同步：

- 交易与案例、评论、标签、策略、保存视图、个人资料、显示偏好。
- 图片、自定义头像与品种图标。
- 快捷键的逻辑动作，以及按平台分别保存的绑定。

仅当前设备：

- 窗口大小与位置、本地库路径、附件缓存状态、更新凭据。
- Windows/macOS 的设备级快捷键覆盖。

## 启动与离线

1. 立即打开本地完整资料库并进入界面。
2. 后台上传 outbox，同时按 cursor 拉取远端增量。
3. 首次登录新设备时先下载元数据检查点，列表可用后继续下载原图。
4. “全部保留离线”完成后，断网仍可查看全部原图。
5. 恢复网络、设备唤醒与定时器触发增量同步。

状态文案必须区分：`本机已保存`、`同步中`、`已同步`、`离线 · N 项待上传`、`N 个冲突待处理`。

## 冲突规则

- 不同字段组自动合并。
- 评论、活动按稳定 ID 并集；标签按增删操作合并。
- 同一篇笔记并发编辑时保留冲突副本。
- 同一笔交易的平仓执行组并发修改时要求用户选择，不静默覆盖。
- 删除墓碑保留至活跃设备确认；长期离线设备需要重新全量引导。
- 恢复旧备份后创建新的 epoch，避免旧云端操作再次覆盖。

## 安全

- 同步凭据仅保存在 Electron 主进程。
- Windows 使用 DPAPI、macOS 使用 Keychain 保存本地密钥。
- 资料库使用随机 256 位密钥；操作与附件使用唯一 nonce 的 AES-256-GCM。
- 新设备通过已登录设备批准或恢复密钥获得资料库密钥。

## 实施顺序

1. 稳定空值、结果来源、实体 ID 与导入规则；停止推荐云盘同步实时库。
2. 增加本地 outbox、deviceId、revision、墓碑与同步状态，不接入远端。
3. 完成单账号双设备的元数据增量同步。
4. 加入附件哈希、加密上传、断点续传和全部离线下载。
5. 加入冲突中心、恢复 epoch、长期离线与故障恢复测试。

## 验收门槛

- 本地首屏不依赖云端，目标 1 秒内可交互。
- 启动后 500ms 内发起同步，普通元数据同步 P95 小于 5 秒。
- 断网启动与编辑成功率 100%。
- 崩溃、重复 push、pull 中断、双设备同时平仓、删除后旧设备上线均不丢数据。
- “全部保留离线”完成后，所有附件通过内容哈希校验并可断网打开。
